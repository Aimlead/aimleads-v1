## Session rules

- Do not change the active ICP during the batch.
- Run the current UI end-to-end without hand-correcting outputs mid-session.
- Record one note immediately after each lead.
- Only treat an issue as real if it repeats at least twice.

## Review cadence

- Review patterns after each batch of 5 leads.
- Look for repeated problems:
  - over-boosting adjacent companies
  - under-scoring strong fits
  - weak or irrelevant signals
  - generic or risky outreach copy
  - UI confusion
  - missing data

## End-of-week decision

- Ship forward only if at least 70% of tested leads feel credible on score, useful on signals, and actionable for outreach.
- If not, fix the top 1 or 2 repeated failure modes before broader testing.
